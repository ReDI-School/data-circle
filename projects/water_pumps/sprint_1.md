# Sprint 1: Exploratory Data Analysis
**Week 1–3**

### Overview
In this sprint, teams will focus on understanding the dataset, cleaning the data, and preparing it for modeling. The goal is to gain insights into water pump functionality patterns and identify potentially important features.

1. Set up project environment, GitHub repository, and documentation  

2. Define your hypotheses. For example:  
- Pump type influences functionality  
- The installer affects pump reliability  
- Geography impacts pump functionality  
- Pump age correlates with failure  

3. Define research questions. For example:  
- Do older pumps fail more often?
- Are pumps serving larger populations more likely to break?
- Are pumps in certain regions or basins more likely to fail?
- How does the installer relate to long-term functionality?
- Which pump types have the highest proportion of functional pumps?

4. Clean and prepare the data. It may include the following steps:
- Validate consistency and data quality
- Remove duplicate entries and entries with missing values for the dependent variable
- Explore outliers
- Reorganise categorical variables with many categories
- Any other steps you find necessary to complete

### Deliverables
- Hypothesis and research questions  
- Key insights and visualizations  
- Reproducible scripts and documentation
- Pitch-presentation (5-minute presentation of results achieved in this sprint)
